# Upstream parser corpus

This directory contains only parser inputs and their text-to-AST outcome. It
does not run or copy upstream minification, transformation, printing, CSSOM,
serialization, source-map, or cascade assertions.

## Lightning CSS

- Version: `lightningcss 1.0.0-alpha.71`, matching the workspace benchmark.
- Sources: `src/lib.rs`, `tests/test_custom_parser.rs`, and
  `tests/test_serde.rs` in the published crate.
- Coverage: every `StyleSheet::parse` call made while running all 117 upstream
  library tests, including inputs generated by loops and `format!`.
- Cases: all 4,223 inputs for which the upstream library parser tests return a
  stylesheet AST. This includes the two custom-at-rule parser cases and the
  serde test's parse phase.
- Excluded: 117 parser-error inputs, because they do not produce an AST.
- Recorded options: error recovery, CSS modules, and parser flags. The rs-css
  harness currently maps the shared error-recovery option and still checks the
  upstream AST-producing outcome for every case.

The original tests were instrumented at the `StyleSheet::parse` boundary. The
result was recorded before the upstream test continued into minification or
printing, so those later phases are not part of this suite.

CLI, bundler, CSSOM, style-attribute, individual property/value, and source-map
behavior is outside this stylesheet/selector AST boundary. Their incidental
parses are therefore not treated as parser corpus cases.

## Stylo

- Version: `stylo 0.19.0` with `selectors 0.39.0`, matching Stylo's dependency.
- Stylo's published crate and standalone mirror contain no stylesheet
  text-to-AST tests. Its parser-related AST tests are in `selectors/parser.rs`.
- Coverage: every statically supplied selector input in those tests, including
  the inputs passed through their local ancestor-hash parse helper.
- Cases: all 80 inputs for which upstream parsing returns a selector AST.
- Excluded: 21 parser-error inputs, because they do not produce an AST.

`corpus.json` retains the upstream test name, source line, expected parse
outcome, input text, and relevant parser option mode for auditability.
